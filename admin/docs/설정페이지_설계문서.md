# 설정 페이지 설계 문서

## 1. 개요

### 1.1 목적
SillyTavern API의 모든 설정을 관리할 수 있는 웹 기반 설정 페이지를 구축한다. 모바일 앱에서는 채팅만 가능하고, 모든 설정은 웹 관리자 페이지에서 관리한다.

### 1.2 범위
- SillyTavern API의 모든 설정 항목 관리
- 설정 카테고리별 탭/섹션 구성
- 설정 저장 및 불러오기
- 설정 검증 및 에러 처리

---

## 2. 설정 구조 분석

### 2.1 설정 카테고리

SillyTavern의 설정은 다음과 같이 분류된다:

#### A. 기본 설정 (General Settings)
- `username`: 사용자 이름
- `user_avatar`: 사용자 아바타
- `active_character`: 현재 활성 캐릭터
- `active_group`: 현재 활성 그룹
- `main_api`: 사용할 메인 API (`openai`, `textgenerationwebui`, `kobold`, `novel`, `horde` 등)
- `swipes`: 스와이프 기능 활성화 여부
- `firstRun`: 첫 실행 여부
- `currentVersion`: 현재 버전

#### B. 생성 파라미터 (Generation Parameters)
- `amount_gen`: 생성할 토큰 수 (최대 응답 길이)
- `max_context`: 최대 컨텍스트 크기 (토큰 수)

#### C. API 제공자별 설정 (Backend API Settings)

**참고**: 이 섹션은 LLM 모델별 설정이 아니라 **API 제공자/백엔드별 설정**입니다.
- `main_api`에서 사용할 API 제공자를 선택 (예: OpenAI, TextGen WebUI, Kobold, NovelAI, Horde)
- 각 API 제공자마다 연결 방식, 인증 방법, 생성 파라미터 등이 다름
- 각 API 제공자 설정 내에서 실제 사용할 **LLM 모델**을 선택함
- 예: OpenAI API 선택 → `oai_settings.model`에서 `gpt-4`, `gpt-3.5-turbo` 등 선택
- 예: TextGen WebUI 선택 → 로컬에서 실행 중인 모델 목록에서 선택
- 같은 모델이라도 다른 API 제공자를 통해 사용할 수 있음 (예: Llama 2를 TextGen WebUI 또는 Kobold로 사용)

##### C-1. OpenAI 설정 (`oai_settings`)
- `api_key`: API 키
- `custom_url`: 커스텀 API URL
- `reverse_proxy`: 리버스 프록시 URL
- `proxy_password`: 프록시 비밀번호
- `model`: 사용할 모델 (예: `gpt-4`, `gpt-3.5-turbo`)
- `temperature`: 온도 (0.0 ~ 2.0)
- `max_tokens`: 최대 토큰 수
- `frequency_penalty`: 빈도 페널티
- `presence_penalty`: 존재 페널티
- `top_p`: Top-p 샘플링
- `stream`: 스트리밍 활성화 여부
- `chat_completion_source`: 채팅 완성 소스 선택
  - `openai`: OpenAI 공식 API
  - `custom`: 커스텀 OpenAI 호환 API
  - `openrouter`: OpenRouter API
  - `vertexai`: Google Vertex AI (Google Cloud)
  - `makersuite`: Google MakerSuite
  - `claude`: Anthropic Claude
  - `perplexity`: Perplexity AI
  - `deepseek`: DeepSeek
  - `ai21`: AI21 Labs
  - `groq`: Groq
  - 기타 등등
- `vertexai_region`: Vertex AI 사용 시 리전 설정 (예: `us-central1`)
- `zai_endpoint`: Zai 엔드포인트 (Zai API 사용 시)

**참고**: Vertex AI는 별도의 API 제공자가 아니라 `main_api`를 `openai`로 설정하고 `chat_completion_source`를 `vertexai`로 선택하면 사용할 수 있습니다. 즉, OpenAI 설정 탭 내에서 소스를 선택하는 방식입니다.

#### D. 고급 설정 (Power User Settings) (`power_user`)

##### D-1. UI/UX 설정
- `show_card_avatar_urls`: 카드에 아바타 URL 표시 여부
- `auto_scroll_chat_to_bottom`: 채팅 자동 스크롤 여부
- `media_display`: 미디어 표시 방식 (`LIST`, `GRID` 등)
- `timestamp_model_icon`: 타임스탬프에 모델 아이콘 표시 여부
- `waifuMode`: 와이푸 모드 활성화 여부

##### D-2. 채팅 관련 설정
- `chat_truncation`: 채팅 잘라내기 (최대 메시지 수)
- `continue_on_send`: 전송 시 자동 계속 여부
- `auto_swipe`: 자동 스와이프 여부
- `auto_continue`: 자동 계속 설정
  - `enabled`: 활성화 여부
  - `target_length`: 목표 길이 (토큰 수)
  - `allow_chat_completions`: Chat Completions API 허용 여부

##### D-3. 프롬프트 관련 설정
- `user_prompt_bias`: 사용자 프롬프트 바이어스
- `show_user_prompt_bias`: 사용자 프롬프트 바이어스 표시 여부
- `encode_tags`: 태그 인코딩 여부
- `allow_name1_display`: 사용자 이름 표시 허용 여부
- `allow_name2_display`: 캐릭터 이름 표시 허용 여부

##### D-4. 마크다운/포맷팅 설정
- `auto_fix_generated_markdown`: 생성된 마크다운 자동 수정 여부

##### D-5. Instruct 모드 설정 (`instruct`)
- `enabled`: Instruct 모드 활성화 여부
- `user_alignment_message`: 사용자 정렬 메시지
- `wrap`: 래핑 여부
- `input_sequence`: 입력 시퀀스
- `output_sequence`: 출력 시퀀스
- `stop_sequence`: 중지 시퀀스

##### D-6. Reasoning 설정 (`reasoning`)
- `prefix`: Reasoning 접두사
- `suffix`: Reasoning 접미사

##### D-7. 캐릭터 관리 설정
- `bogus_folders`: 가짜 폴더 활성화 여부
- `sort_order`: 정렬 순서 (`asc`, `desc`)
- `sort_field`: 정렬 필드
- `aux_field`: 보조 필드 (기본값: `character_version`)

#### E. World Info 설정 (`world_info_settings`)
- World Info 활성화 여부
- World Info 포함 이름 목록 (`world_info_include_names`)
- World Info 앵커 위치 설정 (`wi_anchor_position`)
- World Info 프롬프트 생성 설정
- World Info 깊이(Depth) 설정
- World Info 스캔 설정

**World Info란?**
- 캐릭터나 세계관에 대한 추가 정보를 저장하고 관리하는 기능
- 채팅 중 특정 키워드나 문맥에 따라 자동으로 관련 정보를 프롬프트에 주입
- 캐릭터별로 Primary World와 Auxiliary Worlds를 설정 가능
- API: `/api/worldinfo`

#### F. 확장 기능 설정 (`extension_settings`)

확장 기능은 동적으로 로드되며, 각 확장 기능마다 고유한 설정을 가질 수 있다.

##### F-1. Memory 확장 기능 (`extension_settings.memory` 또는 `extension_prompts['1_memory']`)
- **기능**: 채팅 내역을 요약하여 장기 메모리로 저장
- **설정 항목**:
  - 활성화 여부
  - 요약 주기 (예: N개 메시지마다)
  - 요약 프롬프트 템플릿 (`summarizeString`)
  - 메모리 저장 방식
- **용도**: 긴 대화에서 중요한 정보를 추출하여 컨텍스트에 포함

##### F-2. 벡터 메모리 (Vector Memory)

###### F-2-1. 채팅 벡터 (`extension_prompts['3_vectors']`)
- **기능**: 채팅 메시지를 벡터로 변환하여 의미 기반 검색
- **설정 항목**:
  - 활성화 여부
  - 벡터 모델 설정
  - 검색 결과 개수
  - 유사도 임계값
- **API**: `/api/vector`
- **용도**: 과거 대화에서 유사한 맥락의 메시지를 자동으로 찾아 프롬프트에 포함

###### F-2-2. 데이터뱅크 벡터 (`extension_prompts['4_vectors_data_bank']`)
- **기능**: 외부 데이터베이스나 문서를 벡터로 저장하여 검색
- **설정 항목**:
  - 활성화 여부
  - 데이터뱅크 연결 설정
  - 벡터 모델 설정
  - 검색 결과 개수
- **용도**: 캐릭터나 세계관에 대한 추가 정보를 벡터 검색으로 활용

##### F-3. 기타 확장 기능
- **Floating Prompt** (`extension_prompts['2_floating_prompt']`): 부가 프롬프트
- **ChromaDB** (`extension_prompts['chromadb']`): 벡터 데이터베이스 통합
- **Smart Context**: 지능형 컨텍스트 관리
- 기타 설치된 확장 기능들의 설정

**확장 기능 설정 특징**:
- 확장 기능은 런타임에 동적으로 로드됨
- 각 확장 기능은 고유한 설정 구조를 가짐
- `extension_settings` 객체에 확장 기능별로 중첩된 설정 저장
- `extension_prompts`는 프롬프트 주입을 위한 별도 객체

#### G. 기타 설정
- `tags`: 캐릭터 태그 목록
- `tag_map`: 태그 매핑
- `background`: 배경 설정
- `proxies`: 프록시 프리셋 목록
- `selected_proxy`: 선택된 프록시

## 3. UI/UX 설계

### 3.1 레이아웃 구조

```
┌─────────────────────────────────────────────────┐
│  설정 페이지 헤더                                │
│  [저장] [취소] [초기화]                          │
├─────────────────────────────────────────────────┤
│  [기본] [생성] [OpenAI] [TextGen] [Kobold] ... │  ← 탭 메뉴
├─────────────────────────────────────────────────┤
│                                                 │
│  현재 선택된 탭의 설정 폼                        │
│                                                 │
│  [섹션 1]                                       │
│  - 설정 항목 1: [입력 필드]                     │
│  - 설정 항목 2: [입력 필드]                     │
│                                                 │
│  [섹션 2]                                       │
│  - 설정 항목 3: [입력 필드]                     │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3.2 탭 구성

1. **기본 설정** (General)
   - 사용자 정보
   - 메인 API 선택
   - 기본 생성 파라미터

2. **생성 파라미터** (Generation)
   - `amount_gen`
   - `max_context`
   - 스와이프 설정

3. **OpenAI** (OpenAI Settings)
   - API 제공자: OpenAI (또는 OpenAI 호환 API)
   - **채팅 완성 소스 선택** (`chat_completion_source`):
     - OpenAI 공식 API
     - OpenRouter
     - **Vertex AI** (Google Cloud) - `vertexai_region` 설정 필요
     - Google MakerSuite
     - Anthropic Claude
     - Perplexity AI
     - DeepSeek
     - AI21 Labs
     - Groq
     - 커스텀 API 등
   - API 키 및 연결 설정
   - 모델 선택 (선택한 소스에 따라 사용 가능한 모델 목록이 다름)
   - 생성 파라미터

4. **TextGen** (TextGen WebUI Settings)
   - API 제공자: Text Generation WebUI (로컬 또는 원격 서버)
   - API 서버 설정
   - 모델 선택 (서버에서 실행 중인 모델 목록에서 선택)
   - 생성 파라미터

5. **Kobold** (Kobold AI Settings)
   - API 제공자: Kobold AI (로컬 또는 원격 서버)
   - API 서버 설정
   - 모델 선택 (서버에서 실행 중인 모델 목록에서 선택)
   - 생성 파라미터

6. **NovelAI** (NovelAI Settings)
   - API 제공자: NovelAI
   - API 키 설정
   - 모델 선택 (NovelAI의 모델: clio, kayra, erato 등)
   - 생성 파라미터

7. **Horde** (Horde Settings)
   - API 제공자: AI Horde (분산 컴퓨팅 네트워크)
   - API 키 설정 (선택사항)
   - 모델 선택 (Horde 네트워크에서 사용 가능한 모델 목록에서 선택)
   - 생성 파라미터

8. **고급 설정** (Power User)
   - UI/UX 설정
   - 채팅 관련 설정
   - 프롬프트 관련 설정
   - Instruct 모드 설정
   - Reasoning 설정
   - 캐릭터 관리 설정

9. **World Info** (World Info Settings)
   - World Info 활성화
   - 포함 이름 목록 (`world_info_include_names`)
   - 앵커 위치 설정 (`wi_anchor_position`)
   - World Info 깊이(Depth) 설정
   - World Info 스캔 설정
   - **참고**: World Info는 캐릭터/세계관 정보를 키워드 기반으로 프롬프트에 주입하는 메모리 시스템

10. **확장 기능** (Extensions)
    - 확장 기능 목록 및 활성화/비활성화
    - **Memory 확장 기능**: 채팅 요약 및 장기 메모리 관리
    - **벡터 메모리 (채팅 벡터)**: 의미 기반 메시지 검색
    - **벡터 메모리 (데이터뱅크)**: 외부 데이터베이스 벡터 검색
    - **기타 확장 기능**: Floating Prompt, ChromaDB, Smart Context 등
    - 각 확장 기능별 상세 설정

11. **기타** (Others)
    - 태그 관리
    - 배경 설정
    - 프록시 설정

### 3.4 폼 컴포넌트

재사용 가능한 폼 컴포넌트:
- `TextInput`: 텍스트 입력
- `NumberInput`: 숫자 입력
- `Select`: 드롭다운 선택
- `Checkbox`: 체크박스
- `Slider`: 슬라이더 (온도, top_p 등)
- `TextArea`: 긴 텍스트 입력

## 4. 기능 명세

### 4.1 설정 불러오기
- 페이지 로드 시 `/api/settings/get` 호출

### 4.2 설정 저장

- `/api/settings/save`에 POST 요청
- 성공 시 알림 표시
- 실패 시 에러 메시지 표시

- 저장 전 클라이언트 사이드 검증

### 4.5 실시간 미리보기
- 일부 설정은 변경 시 즉시 미리보기 (예: 배경 설정)

## 8. 참고사항

- SillyTavern의 설정 구조는 매우 복잡하고 동적임
- 일부 설정은 API에 따라 다르게 동작할 수 있음
- 확장 기능 설정은 런타임에 결정되므로 동적 처리 필요
- 설정 저장 시 전체 설정 객체를 보내야 함 (부분 업데이트 불가)

### 8.1 메모리 기능 관련 참고사항

- **World Info**: 캐릭터/세계관 정보를 키워드 기반으로 프롬프트에 주입하는 기본 메모리 시스템
  - API: `/api/worldinfo`
  - 설정은 `world_info_settings`에 저장됨
  - 캐릭터별로 Primary World와 Auxiliary Worlds를 설정 가능

- **Memory 확장 기능**: 채팅 내역을 요약하여 장기 메모리로 저장
  - `extension_prompts['1_memory']`를 통해 프롬프트에 주입
  - 설정은 `extension_settings.memory`에 저장됨
  - 요약 주기와 요약 프롬프트 템플릿을 설정 가능

- **벡터 메모리**: 의미 기반 검색을 통한 메모리 시스템
  - 채팅 벡터: `extension_prompts['3_vectors']` - 과거 대화에서 유사한 맥락 검색
  - 데이터뱅크 벡터: `extension_prompts['4_vectors_data_bank']` - 외부 데이터베이스 검색
  - API: `/api/vector`
  - 설정은 `extension_settings.vectors`에 저장됨
  - 벡터 모델, 검색 결과 개수, 유사도 임계값 등을 설정 가능

- **확장 기능 동적 로딩**: 확장 기능은 런타임에 로드되므로, 설정 페이지에서도 동적으로 처리해야 함
  - 확장 기능 목록은 `/api/extensions` 또는 설정 로드 시 확인 가능
  - 각 확장 기능의 설정 구조는 확장 기능마다 다를 수 있음
