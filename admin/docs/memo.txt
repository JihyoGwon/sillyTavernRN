구체적인 워크플로우 제안
1. 기능 추가 전 확인
너: "이 기능이 뭔지 설명해줘"나: [기능 설명 + 필수성 평가 + 제안 위치]너: "그럼 Phase X에 추가하자" 또는 "나중에 하자"

2. 구현 방식
간단한 기능: 바로 구현
복잡한 기능: 프로토타입 먼저 → 테스트 → 개선

3. 구조 조정
Phase 1-2: 구조 확정 및 유지
Phase 3 이후: 필요 시 구조 조정 (예: "고급 도구" 메뉴 추가)

최종 제안
Phase 1-2: 위계 구조를 확정하고 핵심 기능 구현
Phase 3: 기능 추가 전 간단히 확인 후 구현
Phase 4: Flat 스타일 또는 "고급 도구" 메뉴로 분리
